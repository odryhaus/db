# .BRAND DB Documentation

This folder is intentionally small. Use this page as the map.

## Current Source Documents

- `DATABASE_PLAN.md` — main database architecture and planned additive schema changes.
- `DECISIONS.md` — product and engineering decisions that should not be rediscovered every task.
- `NEXT_STEPS.md` — current deploy checklist and next implementation milestones.
- `KNOWN_ISSUES.md` — active limitations and risks.
- `CRM_SYNC_PLAN.md` — KeyCRM sync architecture and old `/brand-dashboard` analysis.

## Archived Documents

- `archive/AUTH_MVP_PLAN.md` — original authentication MVP plan; historical only.
- `archive/IMPLEMENTATION_REPORT.md` — historical implementation log; not the source of truth.

## Source Of Truth Rules

- Database/table planning belongs in `DATABASE_PLAN.md`.
- Current work queue belongs in `NEXT_STEPS.md`.
- Long-term decisions belong in `DECISIONS.md`.
- Do not keep the same table design in multiple documents.
- Do not treat archived files as current requirements.
